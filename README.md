# Прогнозирование успеваемости студентов по ответам в тесте

Репозиторий содержит **продвинутый baseline** для задачи прогнозирования итоговой успеваемости студентов по их ответам на тестовые задания, а также анализ наиболее значимых признаков.

## Цель проекта

- Построить модель, предсказывающую итоговый результат/успеваемость студента на основе логов ответов в тесте.
- Предложить сильный baseline, который можно улучшать и расширять.
- Проанализировать важность признаков и интерпретировать, какие аспекты поведения студента сильнее всего влияют на прогноз.

## Структура репозитория

- `data.csv` — исходные данные с признаками студентов и их ответами.
- `main.py` — основной скрипт для обучения модели, валидации и вывода метрик.
- `DataCheck.ipynb` — первичный анализ данных, проверка корректности, базовая разведочная аналитика.
- `artifacts/` — сохранённые артефакты экспериментов (модели, метрики, графики важности признаков и т.д.).
- `.python-version`, `pyproject.toml`, `uv.lock` — файлы окружения и зависимостей для воспроизводимости экспериментов.

## Методология

В baseline реализованы типичные этапы ML-пайплайна для табличных данных на основе логов ответов:

- **Предварительная обработка данных**: очистка, работа с пропусками, кодирование категориальных признаков, формирование агрегированных характеристик по студенту.
- **Обучение модели**: прогнозирование успеваемости с использованием градиентного бустинга или других современных алгоритмов для табличных данных.
- **Валидация и метрики**: разделение на train/validation и расчёт метрик качества (RMSE/MAE/R2 для регрессии).
- **Сохранение результатов**: обученная модель и вспомогательная информация сохраняются в `artifacts/`.

## Анализ важности признаков

Дополнительно к обучению модели проводится анализ наиболее значимых признаков:

- Вычисляется важность признаков по обученной модели (feature importance).
- Выявляются характеристики, которые сильнее всего влияют на прогноз успеваемости:
  - поведение в тесте (количество попыток, время на ответы, доля верных ответов),
  - особенности конкретных заданий,
  - агрегированные статистики по студенту.

Этот анализ позволяет использовать модель не только как предсказатель, но и как инструмент для интерпретации и улучшения образовательного процесса.

## Установка и запуск

1. Клонировать репозиторий:
   ```bash
   git clone https://github.com/Igame77/StudentData.git
   cd StudentData
   ```

2. Установить зависимости (через `uv` или `pip`):
   ```bash
   # через uv
   uv sync

   # либо через pip
   pip install -e .
   ```

3. Запустить обучение модели и расчёт метрик:
   ```bash
   python main.py
   ```

После выполнения в каталоге `artifacts/` появятся сохранённые результаты эксперимента.

## Результаты

Baseline модель демонстрирует хорошее качество, учитывая то, что данных для анализа достаточно мало:
- Предсказывать успеваемость студентов с высокой точностью
- Выявлять ключевые факторы, влияющие на результаты обучения
- Использовать полученные инсайты для улучшения образовательного процесса
